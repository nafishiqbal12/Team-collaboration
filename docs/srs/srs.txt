1. Introduction
1.1 Purpose
This document describes the Software Requirements Specification (SRS) for Provider Hub, a
web-based service marketplace platform.
It defines the functional and non-functional requirements of the system and serves as a
reference for development and evaluation.

1.2 Scope
Provider Hub is designed to connect customers with verified local service providers such as:
•

Electricians

•

Plumbers

•

Tutors

•

Cleaners

•

Mechanics

•

IT Support

The system will allow users to search, book, and review services through a centralized and
structured platform.

1.3 Definitions
•

Customer: A user who searches and books services.

•

Provider: A service provider offering services.

•

Admin: System administrator managing users and services.

•

Booking: A service request made by a customer.

•

JWT: JSON Web Token used for authentication.

2. Overall Description

2.1 Product Perspective
Provider Hub is a standalone web application developed using Go (Golang) and MySQL.
It follows a client-server architecture where:
Frontend → Backend (Go + Gin) → MySQL Database

2.2 Product Functions
The system will:
•

Allow user registration and login

•

Enable service searching by category

•

Allow booking of service providers

•

Enable provider verification

•

Allow rating and reviews

•

Provide admin monitoring and reporting

3. System Features
3.1 Customer Module
The system shall allow customers to:

•

Search services by category

•

•

3.2 Provider Module
The system shall allow providers to:
•

Register and create profile

•

Add service details

•

Accept or reject booking requests

•

Update availability status

•

View earnings summary

3.3 Admin Module
The system shall allow admin to:
•

Manage users and providers

•

Verify provider accounts

•

Manage service categories

•

Generate reports

•

Suspend or deactivate accounts
